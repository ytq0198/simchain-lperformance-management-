# 成绩链上存证系统 — 操作说明（虚拟机后端 → 宿主机前端）

本文档说明：**在 Ubuntu 虚拟机中启动 Fabric 网络与后端**，再到 **Windows 宿主机启动前端**并完成联调的推荐顺序。与 **`backend/README.md`**、**`frontend/README.md`**、**`score-chaincode/README-VM.md`** 一致；若与课程讲义冲突，**以课程与老师要求为准**。

---

## 〇、日常启动流程（不修改源码）

**含义**：不改动 **`backend/src`**、**`frontend/src`**、**`score-chaincode`** 等业务源码；仅按环境填写 **`.env`**、**`VITE_API_BASE`**，并在正确目录执行命令。链码已部署且当日无需改链上逻辑时，**不必**重新 **`deployCC`**。

**推荐顺序（背下来即可答辩口述）**

| 步骤 | 位置 | 做什么 | 详见 |
|------|------|--------|------|
| **1** | **虚拟机** | **Fabric**：**`./network.sh up`**（必要时 **`createChannel`**），**`docker ps`** 确认 **orderer / peer** 为 **Up** | **§二** |
| **2** | **虚拟机** | **后端**：**`cd ~/work/backend`**（或你的 **`backend`** 绝对路径） | **§三** |
| **2a** | **虚拟机** | 核对隐藏配置：终端 **`ls -la .env*`** 应能看到 **`.env.example`**；若**没有** **`.env`**，执行 **`cp .env.example .env`** 后再 **`nano .env`**。图形界面里以 **`.`** 开头的文件默认隐藏，按 **`Ctrl+H`** 显示 | **§3.3** |
| **2b** | **虚拟机** | 端口 **3000** 被占用时，结束旧进程（**`Ctrl+C`** 或 **`fuser -k 3000/tcp`**）后再启动 | **§3.8** |
| **2c** | **虚拟机** | **`npm install`**（代码或依赖有更新时；勿从 Windows 拷贝 **`node_modules`**）→ **`npm run build`** → **`npm start`**；须在 **`backend`** 目录内执行，否则 **`.env`** 可能未加载 | **§3.2、§3.4** |
| **3** | **虚拟机** | **自检**：**`GET /api/health`** 与 **`POST /api/auth/login`** 均成功（登录响应须含 **`token`**） | **§3.5** |
| **4** | **Windows** | **前端**：**`frontend/.env.development`** 中 **`VITE_API_BASE=http://<虚拟机IP>:3000`**（勿写宿主 **`localhost:3000`**，除非后端也在本机）；**`npm run dev`**，浏览器打开终端给出的地址（多为 **`http://localhost:5173`**） | **§四** |

**停止与重开**：后端前台运行时 **`Ctrl+C`** 退出，再在同一目录 **`npm start`** 即可；需长期后台见 **§3.8**。

## 一、环境前提（请先确认）

| 位置 | 要求 |
|------|------|
| **虚拟机（Ubuntu）** | 已安装 **Docker**、**Node.js ≥ 18**（`node -v`）；Fabric **`test-network`** 能正常 **`./network.sh up`**；**`score`** 链码已 **commit** 到 **`mychannel`**（参见 **`backend/README.md` 前置条件**）。 |
| **宿主机（Windows）** | 已安装 **Node.js**（建议 LTS）；能 **ping 通虚拟机 IP**；浏览器可访问 **`http://虚拟机IP:3000`**（防火墙 / VMware NAT 已放行 **3000** 端口）。 |
| **代码位置** | 推荐虚拟机内后端路径为 **`~/work/backend`**（与 **`score-chaincode/README-VM.md`** 约定一致）；宿主机为仓库中的 **`frontend/`** 目录。 |

## 二、虚拟机侧：网络与链码（摘要）

以下命令均在 **虚拟机终端** 执行（路径按你本机 **`test-network`** 实际位置修改）。

1. 进入 **`test-network`** 目录，启动网络（若已启动可跳过）：

   ```bash
   cd ~/work/fabric-packages/fabric/fabric/scripts/fabric-samples/test-network
   ./network.sh up
   ./network.sh createChannel   # 若通道已存在，可能提示已存在，以 peer channel list 为准
   ```

2. 确认 **`score`** 链码已部署到 **`mychannel`**（版本以你 **`querycommitted`** 为准）。若尚未部署，按 **`score-chaincode/README-VM.md`** 在链码目录执行 **`go mod tidy`**、**`go mod vendor`** 后再 **`./network.sh deployCC ...`**。

3. **`docker ps`** 中应能看到 **orderer**、**各 org 的 peer** 等容器为 **Up**。

> **注意**：从 Windows **拖拽**代码到虚拟机 **`~/work`** 后，常出现目录**不可写**导致 **`npm install` / `go mod vendor`** 失败。解决办法见 **`score-chaincode/README-VM.md`**：**优先用 Git clone/pull**；若必须拖拽，执行文档中的 **`fixwork`** 或 **`chmod -R u+w ~/work`**，必要时 **`sudo chown -R $USER:$USER ~/work/backend`**。

## 三、虚拟机侧：配置并启动后端（`backend/`）

### 3.1 进入后端目录

```bash
cd ~/work/backend
```

若代码不在上述路径，改为你的 **`backend`** 实际绝对路径所在目录。

### 3.2 权限与依赖（重要，与 README 一致）

| 注意事项 | 操作说明 |
|----------|----------|
| **勿从 Windows 拷贝 `node_modules` 到虚拟机** | 在虚拟机内删除后重装：`rm -rf node_modules && npm install`。否则易出现权限混乱或 **`@grpc/grpc-js`** 与 **`fabric-gateway`** 版本不匹配（参见 **`backend/README.md`** / 实验报告 **§9**）。 |
| **`npm install` / 写日志报 EACCES** | `sudo chown -R $USER:$USER ~/work/backend` 后再执行。 |
| **拖拽后目录只读** | 按 **`score-chaincode/README-VM.md`** 执行 **`fixwork`** 或 **`chmod -R u+w ~/work/backend`**。 |

### 3.3 创建环境变量文件

```bash
cp .env.example .env
nano .env
# 或使用 vim 等编辑器
```

若直接用 **`nano .env`** 而从未执行过 **`cp .env.example .env`**，nano 会显示 **`[ New File ]`** 且内容为空白——表示磁盘上**还没有** **`.env`**，请先 **`cp`** 再编辑。用 **`ls -la .env*`** 可确认：应同时存在 **`.env`** 与 **`.env.example`**（仅有后者时说明尚未复制）。

在 **Ubuntu 文件管理器**中，**`.env.example`** 等以 **`.`** 开头的文件为**隐藏文件**，需按 **`Ctrl+H`**（或菜单「显示隐藏文件」）才能看到。

在 **`.env`** 中必须正确填写（详见 **`backend/README.md` → 配置`**）：

- **`FABRIC_CRYPTO_BASE`**：**Linux 绝对路径**，指向  
  **`…/test-network/organizations/peerOrganizations/org1.example.com`**  
  （即 **`signcerts`**、**`users`** 所在 org1 的 **`peerOrganizations`** 根目录，**不是** Windows 路径）。
- **`JWT_SECRET`**：登录签发 JWT 的密钥，**生产与答辩环境请务必改为随机长字符串**；未设置时后端使用内置弱默认值（仅便于本地调试）。
- **Org2（学生 / 用人单位登录走 Org2 Gateway）**：一般无需单独配置——后端会将 `FABRIC_CRYPTO_BASE` 中的 **`org1.example.com`** 替换为 **`org2.example.com`** 推导 Org2 材料路径。若你机器上 Org2 不在对称路径，可显式设置 **`FABRIC_CRYPTO_BASE_ORG2`**；对等节点默认 **`PEER_ENDPOINT_ORG2=localhost:9051`**、**`PEER_HOST_ALIAS_ORG2=peer0.org2.example.com`**、**`MSP_ID_ORG2=Org2MSP`**（与 **`.env.example`** 一致）。
- 其他项一般保持 **`.env.example`** 默认即可：**`CHANNEL_NAME=mychannel`**、**`CHAINCODE_NAME=score`**、**`PEER_ENDPOINT=localhost:7051`** 等与 **`test-network` + Org1** 一致。

若 **`signcerts`** 下无证书：先在 **`test-network`** 执行 **`./network.sh up`** 生成 **`organizations/`**；cryptogen 常见文件名为 **`User1@org1.example.com-cert.pem`**，本仓库后端会自动解析 **`signcerts/*.pem`**，一般无需手工改名。

### 3.4 编译并启动

```bash
npm install
npm run build
npm start
```

成功时终端出现类似：

```text
score-backend listening on http://localhost:3000
```

**说明**：**`server.ts`** 使用 **`dotenv`** 从 **`backend/.env`**（相对编译输出目录的上一级）加载配置；请在 **`backend` 目录内**执行 **`npm start`**，避免在 **`~`** 等无关目录启动导致 **`.env` 未加载**、**`/api/health`** 中 **`fabricConfigured": false`**（见 **`backend/README.md`**）。

### 3.5 本机自检（仍在虚拟机内）

```bash
curl -s http://localhost:3000/api/health
```

期望 JSON 中含 **`"ok":true`**，且 **`"fabricConfigured":true`**（表示证书路径与 Gateway 初始化正常）。若 **`fabricConfigured` 为 `false`**：检查 **`.env`** 路径、是否在项目目录启动、**`FABRIC_CRYPTO_BASE`** 是否为正确的 **Linux 绝对路径**。

**登录路由自检（与 §〇 步骤 3 一致）**：

```bash
curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"jiaowuchu","password":"demo"}'
```

期望返回 **JSON** 且含 **`token`** 字段。若返回 **HTML**、正文含 **`Cannot POST /api/auth/login`** 或 **HTTP 404**：当前 **3000** 端口上的进程**不是**本仓库带 JWT 的后端（常见为旧代码未更新），请在 **`~/work/backend`** 内 **`git pull`**（或重新同步源码）后执行 **`npm run build`** 并**重启** **`npm start`**，再重试本 **`curl`**。

**说明**：**`/api/health`** 与 **`POST /api/auth/login`** 为**少数无需 JWT** 的接口；其余 **`/api/scores*`**、**`/api/verify`**、**`/api/auth/me`** 均需在请求头携带 **`Authorization: Bearer <token>`**（由登录接口返回）。

### 3.6 登录与演示账号（后端内置）

前端打开应用后会进入**登录页**；登录成功后由后端返回 **JWT** 及用户信息（**`role` / `org` / `attributes`**），并按角色选择 **Org1** 或 **Org2** 的 Fabric Gateway 身份（详见 **`项目设计.md`** 与 **`实验报告.md` §6.3**）。

| 用户名 | 密码 | 角色（链上 Gateway） | 说明 |
|--------|------|----------------------|------|
| `jiaowuchu` | `demo` | 教务处 · Org1 | 可写（录入/更正/作废）、可查、可验真 |
| `teacher01` | `demo` | 教师 · Org1 | 可写、可查、可验真 |
| `student01` | `demo` | 学生 · Org2 | 只读：查询、溯源、**我的证书**；无写菜单、无链上浏览器入口 |
| `hr001` | `demo` | 用人单位 · Org2 | **验真与凭证** + **链上浏览器**；批量验真工具箱 |

链码 **`PutScore` / `CorrectScore` / `RevokeScore`** 在升级后包含 **MSP 级 ABAC**：仅 **Org1MSP** 可写入；学生使用 Org2 证书时，即使绕过 UI 调用写接口，也会在**背书/链码**层被拒绝（答辩可强调「UI + API + 链码」纵深）。**升级链码后**须重新 **`deployCC` 升版** 方可在链上生效。

### 3.7 供宿主机访问：IP 与防火墙

1. 查看虚拟机局域网 IP：

   ```bash
   hostname -I
   ```

   取 **形如 `192.168.x.x` 的地址**作为宿主机浏览器 / 前端配置使用；**不要**用 **`172.17.x.x` / `172.18.x.x`** 这类 Docker 网桥地址作为「虚拟机业务 IP」。

2. 若宿主机无法访问 **`http://虚拟机IP:3000`**：  
   - 检查 **Ubuntu 防火墙**（如 **`ufw`**）是否放行 **3000**：`sudo ufw allow 3000/tcp`（按需）；  
   - 检查 **VMware/VirtualBox NAT** 是否映射或放行 **3000**；  
   - 在 **宿主机 PowerShell** 测试：`curl http://虚拟机IP:3000/api/health`。

3. **写交易较慢**：录入 / 更正 / 作废可能持续数秒至更久，属 **`submitAsync`** 与背书正常行为；前端已用 **NProgress** 提示请求进行中（见 **`frontend/README.md`**）。

### 3.8 后台常驻（可选）

需要离开终端仍运行时，可在 **`backend`** 目录使用：

```bash
nohup npm start > ~/score-backend.log 2>&1 &
```

查看日志：`tail -f ~/score-backend.log`。端口占用时：**`fuser -k 3000/tcp`** 或查找进程后 **`kill`**（**`backend/README.md`** 亦有说明）。

## 四、宿主机侧：配置并启动前端（`frontend/`）

以下在 **Windows** 上操作（路径按你仓库实际位置修改，示例含中文路径）。

### 4.1 进入前端目录

```powershell
cd "C:\Users\你的用户名\...\final-project\frontend"
```

### 4.2 安装依赖（首次或 package.json 变更后）

```powershell
npm install
```

### 4.3 配置 API 地址

编辑 **`frontend/.env.development`**（若无则新建），设置：

```env
VITE_API_BASE=http://你的虚拟机IP:3000
```

示例：`VITE_API_BASE=http://192.168.75.128:3000`  

**说明**：**不要**写成 **`localhost:3000`**（除非你在本机也起了后端）；应写 **虚拟机可从宿主机访问的 IP + 端口 3000**（与 **`backend/README.md` → 与前端联调** 一致）。

保存后需 **重新执行 `npm run dev`** 才会重新加载环境变量。

### 4.4 启动开发服务器

```powershell
npm run dev
```

终端会输出本地访问地址，一般为：

```text
  ➜  Local:   http://localhost:5173/
```

在 **Windows 浏览器** 打开 **`http://localhost:5173`**（端口以终端为准）。这是 **Vite 前端**；**不是** 直接在浏览器打开 **`http://虚拟机IP:3000`**（那是后端 API，只会看到 JSON）。**首次进入会打开登录页**；登录成功后本地保存 JWT，再访问各业务路由。

### 4.5 联调验证建议顺序

1. **登录**：使用上表演示账号（如 **`jiaowuchu` / `demo`**）登录；浏览器本地会保存 JWT，刷新后 **`GET /api/auth/me`** 会校验令牌。  
2. **综合看板（`/dashboard`）**：应能拉到 **`/api/health`**，网关状态与 **`backend`** 一致；**学生**账号会看到个人概况占位提示，**教务处**会看到异常预警占位（演示 UI）。  
3. **成绩查询**：输入学号、课程、学期，与虚拟机带 **`Authorization`** 的 **`curl`** 或 **`peer query`** 对照。  
4. **全链路溯源**：从查询页跳转或手动输入同一键，查看时间轴、Diff 与 **TxID**。  
5. **录入 / 更正 / 作废**：仅 **教务处 / 教师**（Org1）侧栏可见写菜单；提交后查看 **transactionId** 与 **Fabric 流程阶段提示**（背书→排序→提交）。  
6. **验真与凭证**：**教务处 / 教师**可从侧栏进入；**用人单位**登录后默认落在验真页，可使用 **批量验真**（多行 CSV）。**学生**无验真入口（直链 `/verify` 会跳回成绩查询）。  
7. **成绩查询页**：查询成功后可 **导出 PDF 成绩单**、生成指向验真页的 **二维码**（手机扫码可带参打开验真并 **`auto=1`** 自动核验，演示跨端）。  

**使用 `curl` 调 REST（需先登录取 token）**：

```bash
TOKEN=$(curl -s -X POST http://127.0.0.1:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"jiaowuchu","password":"demo"}' | jq -r .token)

curl -s -H "Authorization: Bearer $TOKEN" \
  "http://127.0.0.1:3000/api/scores?studentId=2021003&courseId=PHYS101&semester=2024-3"
```

若未带 **`Authorization`**，接口返回 **401**。学生账号对 **`POST /api/scores`** 会返回 **403**（应用层）且链上亦不接受 Org2 写入（链码层，需已部署含 ABAC 的链码版本）。

后端已启用 **`cors()`**，一般不存在浏览器跨域拦截；若仍有问题，检查 **`VITE_API_BASE`** 是否带错协议或端口。

### 4.6 生产构建（可选）

```powershell
npm run build
npm run preview
```

用于本地预览 **`dist`**；答辩若静态部署，将 **`dist`** 放到 Web 服务器并同样将运行时 **`VITE_API_BASE`**（或等价 **`baseURL`**）指向 **`http://虚拟机IP:3000`**。

## 五、常见问题速查

| 现象 | 处理方向 |
|------|----------|
| **`/api/health` 中 `fabricConfigured":false`** | **`backend/.env`** 未加载或 **`FABRIC_CRYPTO_BASE`** 错误；在 **`~/work/backend`** 下重启 **`npm start`**。 |
| **虚拟机 `curl localhost:3000` 正常，宿主机访问不了** | 防火墙 / NAT / **`hostname -I` 的 IP 选错**（勿用 Docker 网桥 IP）。 |
| **前端一直报错或超时** | **`VITE_API_BASE`** 是否可 **`curl`**；虚拟机 **`npm start`** 是否在跑；**链码与通道**是否已就绪。 |
| **`401` / 前端跳回登录** | JWT 过期或未登录；**`curl`** 未带 **`Authorization`**。重新 **`POST /api/auth/login`** 取 token，或浏览器重新登录。 |
| **学生写入 `POST /api/scores` 返回 403** | 预期行为：Express **`requireRoles`** 拦截；若伪造教师 JWT 而无密钥则无法通过校验。 |
| **Org2 账号报证书路径错误** | **`test-network`** 未生成 **org2** 材料或 **`FABRIC_CRYPTO_BASE_ORG2`** 配置错误；确认 **`organizations/peerOrganizations/org2.example.com/users/User1@org2.example.com`** 存在。 |
| **前端登录提示 `Not Found` / `Cannot POST /api/auth/login`** | **`/api/health` 正常但登录 404**：VM 上后端为旧版，无 **`POST /api/auth/login`**；按 **§〇、§3.5** 更新源码并 **`npm run build`** 后重启。 |
| **拷贝到 VM 后「找不到」`.env.example`** | Linux **隐藏文件**；文件管理器 **`Ctrl+H`**，或终端 **`ls -la`**。 |
| **`nano .env` 显示 `[ New File ]` 且空白** | 尚未 **`cp .env.example .env`**；见 **§3.3**、**§〇**。 |
| **在 bash 里用 `^` 续行导致 `curl` 报错** | **`^`** 为 **Windows PowerShell** 续行符；Linux bash 用 **行尾 `\`** 或写成**一行** **`curl`**。 |
